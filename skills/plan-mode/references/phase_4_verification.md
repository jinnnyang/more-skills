# Phase 4: Verification & Walkthrough

All tasks in `task.md` are marked as `[x]`. It is time to verify the system as a whole and present the results.

## 1. System Verification
- Run the full test suite if one exists (e.g., `npm test`, `pytest`).
- Run the build process (e.g., `npm run build`, `go build`).
- If applicable, start the dev server and verify endpoints or UI locally.
- Fix any remaining bugs. If bugs are complex, add them to `task.md` as new tasks and revert briefly to Phase 3.

## 2. Generate Walkthrough
Create a final report for the user using the `../templates/walkthrough.md` template.
- Summarize what was accomplished.
- List any minor deviations from the original plan.
- Include instructions on how the user can test the new functionality themselves.

## 3. Handoff
Present the `walkthrough.md` to the user. Ask them if they are satisfied or if they want to make further adjustments.
