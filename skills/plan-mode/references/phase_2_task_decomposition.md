# Phase 2: Task Decomposition

The plan has been approved. You must now translate the high-level design into an atomic, actionable checklist.

## Execution Steps

1. **Create the Task Tracker**: Create a `task.md` file in the workspace.
   - You MUST use the structure defined in `../templates/task.md`.
2. **Decompose Atomically**: Break down the approved plan into discrete steps.
   - Good: `[ ] Create User API route in src/routes/user.js`
   - Bad: `[ ] Build the backend`
3. **Order by Dependency**: Ensure tasks are ordered logically. E.g., create database models before creating API routes that rely on them.

Once the `task.md` is populated, immediately proceed to **Phase 3: Execution**. You do not need to wait for human approval for the task list unless you are unsure about the implementation order.
