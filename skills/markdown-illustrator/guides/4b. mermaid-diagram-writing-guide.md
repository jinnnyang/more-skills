# 📊 Mermaid 逻辑图表编写与美化指南 (Mermaid Chart Writing Guide)

本指南旨在指导 AI Agent 或排版编辑将文章中的结构化、逻辑化、流程化信息，通过 **Mermaid 源码文件 (`.mmd`)** 的形式提取为高质量的视觉插图。通过对接文章的全局风格与色盘，确保逻辑图表不再具有“粗糙的黑白默认感”，而是成为整篇文章视觉艺术系统的一部分。

---

## 🛠️ 执行步骤概览

```mermaid
    %% flowchart TD：声明图表类型为流程图，并且全局方向为 Top-Down (从上至下)。其他方向可选：LR(从左到右), RL(从右到左), BT(从下到上)
    flowchart TD
    %% classDef：预先定义类的样式，用于后续为节点绑定特定颜色和粗细。可以使用 CSS 颜色名称或十六进制代码
    classDef default fill:#FAFAF8,stroke:#1A2530,stroke-width:1px,color:#1A2530;
    classDef highlight fill:#FAFAF8,stroke:#C62828,stroke-width:2px,color:#C62828,font-weight:bold;
    
    %% A["..."]：定义一个直角矩形节点A，并赋文本。双引号强制要求用于包裹可能含标点的文本。节点ID (如 A, B) 必须是字母或数字
    %% -->：定义一条单向实线连接箭头。增加横线数量可以延长连线长度（如 ---> 会比 --> 长），从而增加节点之间的间距
    %% B{"..."}：定义一个菱形判断节点B，常用于表示条件分支
    A["开始：AI 代理阅读段落语义"] --> B{"是否包含流程/时序/结构信息?"}
    %% -->|"..."|：定义一条附带中心描述文本的连线。文本同样需要用双引号包裹以避免标点符号导致的语法错误
    %% :::highlight：为特定节点应用之前通过 classDef 定义的样式类
    B -->|"是 (提取流向)"| C["选择合适的 Mermaid 图表类型"]:::highlight
    B -->|"否 (普通文本)"| D["放弃配图，继续阅读"]
    
    C --> E["依据安全语法编写 .mmd 源码<br/>(必须使用双引号包裹外显文本)"]
    %% -.->：定义一条单向虚线连接箭头。类似地，增加点和横线（如 -..->）可以拉长虚线距离
    E -.-> F("调用 scripts/mermaid-tool.ts<br/>进行本地语法校验")
    F --> G{"校验是否通过?"}
    G -->|"通过"| H["输出 Markdown 相对路径引用"]:::highlight
    G -->|"失败"| I["查询最新官网文档进行自愈修复"]
    I -.-> E
    
    %% linkStyle default：全局统一设定默认所有连线的样式风格（颜色、粗细、线型等）
    linkStyle default stroke:#1A2530,stroke-width:1px;
    %% linkStyle 2,6：通过从0开始的编号（即图表声明中的第3条和第7条线），单独指定这些连线的样式。非常适合用于高亮特定的路径
    linkStyle 2,6 stroke:#C62828,stroke-width:2px;
```

## 📖 详细步骤指导

### 步骤一：分析段落语义，进行图表选型 (Semantic Selection)
在视觉图纸（Spec）规划阶段，根据段落的语言特征和逻辑类型，对照下表选择最合适的 Mermaid 图表类型（严禁一律使用 Flowchart）：

| 逻辑特征 | 推荐图表类型 | 英文关键字 | 说明与适用场景 |
| :--- | :--- | :--- | :--- |
| **步骤、因果、决策分支、拓扑结构** | 流程图 | `flowchart TD` 或 `flowchart LR` | 表达行动指南、操作流程、系统组件拓扑。 |
| **多主体间的消息传递与时序流向** | 时序图 | `sequenceDiagram` | 接口调用、系统服务交互、人机对话过程。 |
| **历史演进、关键里程碑、时间脉络** | 时间线 | `timeline` | 发展历程、项目阶段总结、历史大事件盘点。 |
| **项目进度排期、工期依赖、资源冲突** | 甘特图 | `gantt` | 项目里程碑进度、研发排期、阶段耗时展示。 |
| **多维度定位、权重归类、战略评估** | 四象限图 | `quadrantChart` | 任务优先级划分（如价值 vs 难度）、产品定位。 |
| **数值流转分配、预算去向、能量损耗** | 桑基图 | `sankey-beta` | 资金分流、资源去向。**注意：标签严禁使用双引号。** |
| **趋势变化与类别数量对比** | XY 图表 | `xychart-beta` | 月度数据趋势、横向柱状对比（可组合折线与柱状图）。 |
| **多维属性素质分析、竞争力雷达** | 雷达图 | `radar-beta` | 候选人素质评估、多维度对比。**（需要 Mermaid v11+，本地不支持校验）** |
| **版本控制、代码分支、合并演进** | Git 提交流 | `gitGraph` | 代码提交流程说明、分支合并模式演示。 |
| **集合关系、交集并集、互斥对比** | 韦恩图 | `venn-beta` | 概念重叠区域、受众交集分析。**（需要 Mermaid v11+，本地不支持校验）** |
| **核心概念发散、多层树状解构** | 思维导图 | `mindmap` | 知识结构图、大纲大类拆解、脑图。 |

---

### 步骤二：读取 Spec 定义，提取色票与风格 (Reference Mapping)
**【禁止硬编码自定义颜色】**。你必须读取当前文章规划 spec (`[文件名].plan.md`) 中的颜色配置，直接映射到 Mermaid 的样式类定义（`classDef`）中：

1. **色盘映射关系**：
   * **`Background` (底色)** $\rightarrow$ 用于图表底层或常规节点底色。
   * **`Primary` (主色)** $\rightarrow$ 用于常规节点的线条边框与文字。
   * **`Accent` (点缀色)** $\rightarrow$ 用于核心/重点/特殊节点的高亮（填充底色或粗边框）。
   * **`Soft Fill` (浅填充)** $\rightarrow$ 用于辅助区域或次要节点的浅色填充。

2. **视觉流派映射 (以 Flowchart 为例)**：
   * **瑞士理性风 (Swiss)**：
     * 节点边框必须为直角（仅使用 `NodeID["Label"]`），严禁使用圆角。
     * 连接线必须平直（必须使用 `flowchart` 关键字，而非旧版 `graph`，以保证渲染出直角连线）。
     * 禁用渐变、阴影和 Emoji。
   * **Notion 极简手绘风 (Notion)**：
     * 节点边框使用圆角（使用 `NodeID("Label")`）。
     * 连线推荐使用虚线（如 `-.->`）表现轻量感。
     * 整体采用黑、白、灰色调，仅使用一个柔和的 Soft Fill 进行重点标记。

### 步骤三：语法安全防线与引号规则 (Syntax Guardrails)
Mermaid 的解析器非常脆弱，任何标点冲突都会导致渲染报错崩溃。必须严格遵守以下防错规则：

1. **对外展示的文本必须强制使用双引号包裹**：
   * ❌ *错误*：`A --> B(验证条件:成功?)` （冒号和括号会直接引发编译器崩溃）
   * ✓ *正确*：`A --> B("验证条件:成功?")`
2. **换行与富文本处理**：
   * 双引号内的内容会送入 Markdown 渲染器解析。
   * **换行**：请统一使用 `<br/>` 标签进行换行（如 `Node["第一行文本<br/>第二行文本"]`）。
   * **文本强调**：如果需要粗体，使用 Markdown 格式（如 `Node["**核心高亮**"]`）。

## 📐 三、 语法解析过程与安全防线 (Syntax Parsing & Guardrails)

为了让你直观理解为什么 Mermaid 对标点如此敏感，下面用**状态图 (State Diagram)** 解释 Mermaid 的语法解析生命周期。请将此图作为状态图的**自解释示例**。

```mermaid
    %% stateDiagram-v2：声明图表类型为状态转移图的最稳定大版本。v2 版本提供了更好的布局算法和渲染效果
    stateDiagram-v2
    %% 状态图自解释示例：展示 Mermaid 语法解析器的脆弱性
    %% [*]：这是一个特殊语法，代表状态图中的初始起点（实心黑点）或最终结束点（带圆圈的黑点）。在连线左侧代表起点，右侧代表终点
    %% -->：代表状态之间的转移连线。在状态图中，连线通常只用 -->，不支持像流程图中那样随意增减长度
    [*] --> 词法分析器Lexer

    %% state ID { ... }：定义一个复合型状态容器（Composite State），其内部可包含自己独立的子层级状态流转。ID 可以是任意合法的非空字符串
    state 词法分析器Lexer {
        [*] --> 扫描关键字
        %% 状态A --> 状态B : 冒号后面紧跟发生该状态转移的触发条件或说明文本。该文本会显示在连线上方
        扫描关键字 --> 捕获节点文本 : 遇到 [" 或者 ("
        
        state 捕获节点文本 {
            %% direction LR：单独设定该状态容器内部的节点排版方向从左至右 (Left to Right)。默认方向是自上而下 (Top to Bottom)
            direction LR
            安全文本 --> 成功解析 : 纯汉字或纯英文
            危险符号 --> 崩溃报错 : 包含括号冒号等未转义危险符号
        }
    }

    词法分析器Lexer --> 抽象语法树AST : 构建成功
    词法分析器Lexer --> [*] : Parse Error 💥 (例如忘记用双引号包裹含标点的文本)
    
    抽象语法树AST --> 渲染引擎Renderer
    渲染引擎Renderer --> [*] : 生成最终 SVG 图片
    
    %% note right of ID: 在某个指定状态节点的右侧附加一张黄色的文字注释便签贴。也可以使用 `note left of`。换行同样依赖于 HTML 标签 <br/>
    note right of 词法分析器Lexer: 任何在节点定义中暴露的<br/>特殊标点都会导致 Lexer 判定为非法 Token
```

> [!CAUTION]
> **语法防错核心结论**：
> 1. **对外展示的文本必须强制使用双引号包裹**：`A --> B("验证条件:成功?")`
> 2. **换行**：请统一使用 `<br/>` 标签进行换行。
> 3. **缩进**：不要在关键字（如 `sankey-beta`）后面或前面随意加空格，换行后顶格写或遵循2空格缩进即可。

### 2. Sequence Diagram (时序图)

```mermaid
    %% sequenceDiagram：声明图表类型为时序交互图。用于按时间顺序展示系统或角色间的消息交互过程
    sequenceDiagram
    %% autonumber：自动为所有的交互动作箭头依次打上数字序号（例如 1, 2, 3...）。方便在正文中引用具体交互步骤
    autonumber
    %% actor：声明一个“操作者/人”角色的参与实体（渲染为人形状的小图标），并用 as 设置页面显示的中文别名
    actor U as "普通党员"
    %% participant：声明一个普通的“参与方/系统”实体节点（渲染为矩形块）。节点在图纸中会按此处声明的顺序从左到右排列
    participant B as "党支部委员"
    participant D as "上级党组织"
    
    %% ->>：实线箭头，通常表示同步调用、主动发送动作或业务请求流。
    %% 其他箭头类型：->（无箭头实线）, -x（带叉实线，表示异步/丢失的消息）
    U ->> B : "提交单点突破学习报告"
    B ->> D : "汇报支部学习地图拼图进度"
    %% -->>：虚线箭头，通常表示异步回调、处理完成的数据响应或下发结果。
    D -->> B : "下发阶段性学习辅导材料"
    B -->> U : "分发定制版精读笔记"
    %% 高级特性提示：可以使用 rect rgb(200, 200, 200) ... end 来绘制背景矩形框以高亮某段特定的交互流
    %% 还可以使用 activate B 激活某个生命周期（显示为竖向矩形），用 deactivate B 结束
```

### 4. Gantt Chart (甘特图)

```mermaid
    %% gantt：声明图表类型为甘特图（项目排期视图），用于展示随时间推进的项目进度、里程碑及依赖关系
    gantt
    title "党建宣贯工作第一阶段排期表"
    %% dateFormat：指定后续任务写入排期日期的解析格式（如 YYYY-MM-DD）。系统会依据此格式解析开始和结束时间
    dateFormat YYYY-MM-DD
    %% axisFormat：指定图表顶部横坐标时间轴最终呈现的格式标签（如 %m-%d 意为月-日，%Y-%m 为年-月）
    axisFormat %m-%d
    
    %% section：对下属的任务条进行纵向的大类分组区块划分。同一个 section 内的任务会归为一组展示
    section 自学阶段
    %% 任务名 : 状态标签(可选), 任务唯一ID(可选), 开始时间/依赖项, 结束时间/持续时长
    %% 状态标签：active(正在进行中), done(已完成), crit(关键路径节点)，如果不填则默认为未开始状态
    %% 时长单位：d(天), w(周), h(小时), m(分钟)
    精读自学 : active, a1, 2026-06-01, 7d
    %% after a1：表示此任务强依赖于指定 ID (a1) 的任务，将在 a1 完成的瞬间即时开始
    撰写心得 : after a1, 3d
    
    section 集中阶段
    专题党课准备 : b1, 2026-06-05, 3d
    %% 高级特性提示：里程碑任务可以添加 milestone 标签，表示时间点： 里程碑任务 : milestone, m1, 2026-06-08, 0d
    支部集中研讨 : after b1, 2d
```

### 7. XY Chart (XY图表)

```mermaid
    %% xychart-beta：声明图表类型为多维数据 XY 图表（可混排柱状图 bar 与折线图 line）

    x-axis ["党群部", "技术部", "销售部", "运营部", "后勤部"]

    y-axis "参会率 (%)" 0 --> 100
```

### 8. Radar Diagram (雷达图)

```mermaid
    %% radar-beta：声明图表类型为多维属性雷达对比图（蛛网图）。非常适合做能力素质模型、产品多维竞争力评估
    radar-beta
    title "两会学习贯彻成果多维度成效分析"
```

## 🛠️ 五、 校验、输出与交付规范 (Validation & Delivery)

### 1. 本地语法安全校验
在向工作区保存图表之前，你必须使用本地的 `mermaid-tool.ts` 脚本对代码进行语法验证：
```bash
npx tsx scripts/mermaid-tool.ts --validate-only --code "YOUR_MERMAID_CODE"
```
> [!WARNING]
> **版本局限性处理**：
> 本地开发环境的 `pretty-mermaid` CLI 使用的是 Mermaid v10.9.0，因此无法识别 `radar-beta` 与 `venn-beta`。
> *   对除了雷达图和韦恩图之外的 **9 种图表**，如果验证返回错误，必须立即根据诊断日志修复代码，切勿将破损的语法写入文件。
> *   对 **雷达图 (`radar-beta`)** 与 **韦恩图 (`venn-beta`)**，在确定语法符合最新官方标准的情况下，**可以跳过本地校验直接保存为 `.mmd`**，下游渲染管线会自动处理。

### 2. 后缀名与文件命名规范
*   **后缀名要求**：所有的 Mermaid 图表必须以原生的 **`.mmd`** 源码文件进行物理保存（下游将有渲染管线统一处理），绝对不能保存为 `.svg` 或 `.png`。
*   **物理保存路径**：默认保存在正文同级目录下的 `./images/` 中。
*   **命名公式**：`[article-slug]-[content-focus]-chart.mmd`

### 3. Markdown 中的引用与交付输出 (统一 I/O 契约)
图表生成并校验完毕后，**不要修改用户的正文 MD 文件**。Agent 必须将产出结果以单行标准格式**追加写入**到原始文章同级目录下的 `[原文章名].cand.md` 清单文件中：

```markdown
<建议插入行号>: ![<这里写入高度详细的图表结构描述，供屏幕阅读器朗读>](images/[三段式文件名].mmd "<简明扼要的标题>")
```

*示例*：
```markdown
105: ![这是一张从上至下的工作流图表。起于“两会闭幕”，流向一个判断节点“是否有分类指导方案”，若“有”则流向高亮节点“分层级、分群体学习”并最终指向“达成全景穿透效应”；若“没有”则流向“泛读大水漫灌”并指向“形式主义”。核心重点分支连线采用红色加粗显示。](images/lh-learning-overall-process-chart.mmd "两会学习贯彻全景穿透工作流")
```

同时可以在对话中向用户展示源码，但写入 `*.cand.md` 是自动装配的前置动作。

## 🚨 六、 语法容灾与自愈协议 (Error Recovery Protocol)

由于 Mermaid 的图表语法在不断演进（例如带有 `-beta` 标识的图表类型未来可能会变更为正式版），如果你在调用本地 `scripts/mermaid-tool.ts` 进行语法校验时，遇到解析报错（例如：`Invalid Mermaid code...`）：

> [!CAUTION]
> **绝对不允许猜测语法**。你必须立刻使用 **网页内容获取工具** 访问以下官方文档路径查阅最新的语法标准与关键字：
> *   官方语法主入口：[https://mermaid.ai/open-source/syntax/](https://mermaid.ai/open-source/syntax/)
> *   具体图表子页面，例如：
>     *   `https://mermaid.ai/open-source/syntax/flowchart.html`
>     *   `https://mermaid.ai/open-source/syntax/radar.html`
>     *   `https://mermaid.ai/open-source/syntax/sankey.html`
> 
> 读取最新的官方语法，修复本地代码，并重新调用 `scripts/mermaid-tool.ts --validate-only` 进行校验，直至完全通过。
