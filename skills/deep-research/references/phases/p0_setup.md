# Phase 0: 环境准备与交互式需求确认 (Setup & Intake)

本阶段指导主智能体（Lead Agent）检查运行环境、主动探测当前就绪的工具与技能，并在交互式 Intake 中向用户显式列出，最后初始化项目根目录。

---

## 1. 前置环境与依赖核对 (Environment Check)

在进入任何任务前，必须自动执行检测脚本：
```bash
python scripts/check_env.py
```
* **核心阻断项**：若网络连接全部失败、或当前工作目录没有写权限，应立即报错停止，并提示用户。
* **非阻断项**：若 Git 未安装，打印 Warning 警告“本地 Git 未可用，将自动跳过版本追踪提交”，但**绝不能阻断**技能继续推进。

---

## 2. 可用工具与技能主动探测 (Tool & Skill Discovery)

在与用户进行 Intake 沟通前，Lead Agent **必须主动执行 Progressive Discovery 扫描**，并在对话中显式列出探测结果，杜绝黑盒执行：

1. **MCP 工具探测**：
   - 检查已注册的 mcp 工具（例如 `google_search` / `bing_search` / `europepmc` / `read_url_content` 等检索和爬取工具）。
2. **工作区已就绪技能探测**：
   - 检查当前工作区目录下已安装的兄弟技能（读取 `skills/` 目录下的文件夹，例如 `fact-checker` 事实核对、`clinical-trials-database` 临床试验、`pubmed-database` 医学文献、`ensembl-database` 基因库等）。
3. **绘图与可视化工具探测**：
   - 检测是否有注册的 `matplotlib`、`canvas-design` 或其他绘图相关工具，为后续 P3/P4 绘图决策做准备。

---

## 3. 交互式 Intake 需求确认与主动提示

Lead Agent 基于用户的初始研究诉求及上述探测结果，自动向用户提出一个**选择题交互模板**，严禁使用填空题。

### 主动提示与交互模板：
主智能体向用户汇报的信息必须包含以下三个板块：

```markdown
您好！我已经对当前环境的工具和技能进行了主动探测，以下是本次深度调研的就绪配置：

### 🔍 已发现并就绪的工具与技能：
- **基础检索与抓取**：{例如：bing_search, read_url_content}
- **专属领域辅助技能**：{例如：检测到 pubmed-database (医学文献库)、clinical-trials-database (临床试验) 兄弟技能已就绪，将在对应子任务中自适应调用}
- **可视化/质量工具**：{例如：检测到 fact-checker (事实核对) 技能已就绪，拟在 P7 阶段作为最终质量审计；检测到无专门绘图技能，图表将自动回落为内联 Mermaid 语法}

---
```

## 4. 项目初始化与目录构建

用户确认选择其中一个配置方案（如项目命名为 `[project-name]`）后，在工作区下创建该项目的专用根目录：
1. **Git 初始化**：在根目录下运行 `git init`。如果检测到 Git 不可用，跳过该步。
2. **创建子文件夹**：
   - `assets/`（用于存放图表、多媒体佐证的相对引用文件）
   - `findings/`（用于存放各子任务的调研 notes 原始文件）
   - `tasks/`（用于存放任务分配描述）
   - `plan/`（用于存放调研计划与大纲）
   - `kanban/`（用于存放看板与状态）
3. **写入主说明文件**：
   - 写入基础 `README.md` 与 `README_zh-CN.md`。

---
