# Phase 0: 环境准备与交互式需求确认 (Setup & Intake)

本阶段指导智能体检查运行环境、与用户确认调研边界并初始化项目目录。

---

## 1. 自动化环境核对 (Environment Check)
启动前，必须运行检测脚本：
```bash
python scripts/check_env.py
```
- **核心阻断项**：若网络连接全部失败、或当前工作目录没有写权限，应立即报错停止，并提示用户。
- **非阻断项**：若 Git 未安装，打印 Warning 警告“本地 Git 未可用，将自动跳过版本追踪提交”，但**绝不能阻断**技能继续推进。

---

## 2. 交互式 Intake 需求确认
基于用户的初始研究诉求，自动提炼出 **3 个具体候选选项**供用户单选或多选确认，杜绝填空题式提问。

### 候选内容必须涵盖：
1. **研究主题**（具体聚焦的行业、政策或技术命题）
2. **研究范围与边界**（时间跨度、地域限制、核心排除领域）
3. **项目根目录命名**（短横线命名法，如 `rare-earth-export-control-study`）

---

## 3. 项目初始化与目录构建
用户确认首选命名为 `[project-name]` 后，在工作区下创建该项目的专用根目录：
1. **Git 初始化**：在根目录下尝试运行 `git init`。如果检测到 Git 不可用，跳过该步。
2. **创建子文件夹**：
   - `assets/`（用于存放图表、多媒体佐证的相对引用文件）
   - `findings/`（用于存放各子任务的调研 notes 原始文件）
   - `tasks/`（用于存放任务分配描述）
   - `plan/`（用于存放调研大纲与边界文档）
   - `kanban/`（用于存放看板与状态）
3. **写入主说明文件**：
   - 写入基础 `README.md` 与 `README_zh-CN.md`，记录项目元数据与核心目标。

---
