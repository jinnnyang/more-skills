# Deep Research (深度调研工程化技能说明文档)

`deep-research` 是 Antigravity 智能体框架中的核心调研与情报合成技能，用于生成格式受控、证据可追溯、来源可治理且具备多维交叉验证的高质量研究报告。

本技能已全面升级为**工程化持续分阶段调研版本**，支持跨会话标准 Markdown 看板管理、多媒体原始 URL 数据引流、可视化工具发现与 Mermaid 图表回落机制，并规范了启动前的前置环境检查与严格的反模式防护控制。

---

## 🛠️ 核心能力

1. **工程化项目管理**：引入独立项目目录与 Git 版本追踪。每个调研课题拥有一个独立的 kebab-case 格式的根目录（如 `rare-earth-export-control-study`）。
2. **环境工具与专属技能主动探测**：在 P0 阶段自动扫描就绪的 MCP 工具与工作区兄弟技能（如 `pubmed-database`、`fact-checker` 等），并在需求确认（Intake）交互中主动向用户显式列出，提供清晰的分析链路就绪状态。
3. **自适应多层递归深挖**：在 P1 规划与 P2 子任务执行中引入宽度（Breadth）与深度（Depth）控制。子代理可根据搜索出的结论线索使用 CoT 推导，自适应向下递归检索，并在执行中自适应调用专属技能处理特定领域子任务。
4. **状态看板持续追踪**：通过 `kanban/KANBAN.md` 实时记录和更新看板阶段、子任务分配（包含递归参数）、已校验引文列表及待撰写章节进度，支持调研中断与跨会话恢复。
5. **多媒体佐证 URL 规范**：子代理提取网页原图 URL，Lead Agent 在 P3 阶段统一完成图片过滤、Alt 说明审计和引文链接重写，不下载二进制文件，避免防盗链下载报错。
6. **图像 Alt 说明 LLM 友好化**：采集的多媒体佐证必须配备极其详尽的 `alt` 数据及趋势说明（Markdown 图片格式：`!["alt"](url "title")`，且 alt 和 title 必不可少），帮助下游其他 LLM 在阅读报告时完美理解图片信息。
7. **可视化绘图回落策略**：优先检测并调用注册的绘图技能输出 PNG 保存至相对路径 `assets/`；无专门绘图技能时，自动回落为编写防错的 Markdown 内联 Mermaid 代码块。
8. **自动化环境检查**：集成 `check_env.py`，自动测试网络延时、Git 可用性、目录写权限及必要依赖。缺失 Git 或局部网络超时时仅作为 Warning 警告，绝不报错阻断。
9. **大文本流式追加与防截断**：将 P5 阶段重构为分章节循环生成并流式追加（Loop-Append）写入。若发生中途截断，支持从看板断点恢复续写，并对子代理输出设定严格的 token 预算，彻底解决 `length` 截断导致的工具崩溃问题。
10. **工作区作为长期记忆**：引入扁平化共享文件夹（`memories/`、`findings/`、`clues/`、`hypotheses/`），主代理与子代理通过向这些文件夹记录线索与发现，并周期性同步，主动抛弃旧的页面抓取上下文，从而彻底解决超大型复杂调研中的上下文溢出问题。

## 📐 核心设计模式与流程控制

### 1. 交互式 Intake 确认机制
在技能正式启动前，Lead Agent 会基于用户的研究诉求，自动推荐 **3 个左右的具体可选项**供用户单选或多选确认，杜绝填空题式提问。候选包含：
- **研究主题**候选（Topic）
- **研究范围与边界**候选（Boundary）
- **项目根目录命名**候选（Project Name）

### 2. 3 阶段 Git 自动提交生命周期
为确保长周期复杂调研的数据安全性与干净的历史树，技能在以下 3 个关键节点执行 Git 自动提交：
- **📌 [Git 自动提交 1/3]**：`stage: plan-initialized`（P1 阶段完毕，任务板与 Kanban 看板规划完成）
- **📌 [Git 自动提交 2/3]**：`stage: draft-report-written`（P5 阶段完毕，报告初稿分章节追加拼接完成）
- **📌 [Git 自动提交 3/3]**：`stage: report-finalized`（P7 阶段完毕，完成反方审查纠错，引文与 README 结项归档完成）

## 📁 目录结构

当您新建一个调研项目时，技能将自动在您指定的目录下创建如下规范化工程：

```
[project-name]/               # 项目根目录 (kebab-case)
  ├── .git/                  # Git 版本库
  ├── assets/                # 本地化保存的绘图图片或多媒体素材相对路径
  ├── memories/              # 存放历史背景与知识库记忆 (task-*.md)
  ├── findings/              # 存放子代理执行各任务返回的客观事实与核心发现 (task-*.md)
  ├── clues/                 # 存放未经验证的线索或有价值的探索方向 (task-*.md)
  ├── hypotheses/            # 存放智能体根据数据推演出的独立判断与假设 (task-*.md)
  ├── tasks/                 # 存放具体的任务定义、分配及过程记录 (task-*.md 的配置)
  ├── plan/                  # 存放调研计划、大纲及边界文件
  ├── kanban/                # 看板数据存放目录
  │     └── KANBAN.md        # 看板管理、已校验引文列表与多媒体注册表
  ├── README.md              # 项目总体说明（英文，主入口）
  └── README_zh-CN.md        # 项目总体说明（中文，主入口）
```

## 🚀 使用指南

### 1. 执行前置环境校验
Lead Agent 会在 P0 启动前自动执行该检查，您也可以手动在 `deep-research` 技能根目录下调用以排查依赖：
```bash
python scripts/check_env.py
```
*注：该脚本将测试对常用检索站点（如 Bing、EuropePMC）的连接，验证 Git 是否可用以及是否具有工作区写权限，去除了 PIL 等冗余依赖。*

### 2. 启动全新调研项目
当您输入“帮我调研一下...”或“深度研究...”时：
1. 技能将启动 Interactive Intake 并为您提供 3 个项目命名及范围的建议。
2. 选定后，技能将在指定目录下自动创建并初始化 Git。
3. 动态规划任务看板并记录在 `kanban/KANBAN.md`。

### 3. 恢复或迭代已中断的调研
如果运行意外中断，技能再次启动时会检查本地是否存在 `.git` 和 `kanban/KANBAN.md`：
1. 自动载入已完成的任务 Findings。
2. 仅对 `TODO` 状态的子任务进行派发。
3. 无缝衔接后续的引文数据清洗、报告分章节流式合成与反方审查阶段。
