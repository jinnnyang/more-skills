# Guide 1: Markdown 输入标准化与图片补全 (Normalize Markdown Inputs)

本指南是**风格蒸馏的第一个必经步骤**，负责对输入的原始 Markdown 文件进行标准化预处理，确保后续风格分析的数据质量。

---

## 预处理目标

1. **图片完整性补全**：对所有缺失 Alt 或 Title 的图片，通过多模态 LLM 分析图片内容，自动补全描述性的 Alt 文本和图注。
2. **基础格式对齐**：
   - 自动添加中英文混排空格
   - 统一引号为规范中文标点
   - 推断缺失的代码块语言标记
3. **输出标准化文件**：预处理后的文件保持原文语义和链接不变，仅补全缺失信息和对齐格式。

---

## 执行步骤

### 1. 依赖检查
确保环境满足：
- Node.js 可用
- `pretty-markdown/scripts` 目录已运行 `npm install`
- `pretty-markdown/scripts/.env` 文件存在且配置了有效的多模态 LLM API：
  ```env
  OPENAI_API_KEY=your_api_key
  OPENAI_BASE_URL=https://api.openai.com/v1  # 可选，自定义端点
  OPENAI_MODEL_NAME=gpt-4o                 # 必须是支持多模态的模型
  ```

### 2. 调用专门的标准化脚本
使用 `writing-style` 提供的简化脚本，仅专注于补全图片信息和格式标准化：
### 2. 调用标准化脚本
```bash
cd C:\Users\jinnn\Documents\more-skills\skills\writing-style\scripts && node cli.js <input-file.md> [output-file.md]
```
- 如果省略 `[output-file.md]`，将**原位覆盖**输入文件
- 如果指定输出文件，原始文件将保留不变
- 依赖会自动安装，如果不存在 `node_modules`
- 如果指定输出文件，原始文件将保留不变

### 3. 执行后检查
脚本执行完成后：
- 检查退出码是否为 0
- 如果有错误提示，按照提示解决（通常是 API 密钥缺失或网络问题）
- 预处理完成后，进入下一阶段风格蒸馏（现在从 `2. style-distillation-guide.md` 开始）

---

## 设计说明

为什么把这一步单独拆出来？
1. **职责分离**：标准化是独立的前置工程任务，不混入风格分析
2. **可复用性**：标准化后的 Markdown 可以供给任意下游任务使用
3. **可调试性**：如果图片补全出错，可以单独调试这一步
4. **保持纯净**：不改变原文内容，只补全缺失信息，不影响风格提取准确性

---

## 关键规则

- ✅ 只补全**确实缺失**的 Alt/Title，已有完整信息的图片保持原样不覆盖
- ✅ 不改变原始图片链接，远程链接保持远程，本地链接保持本地
- ✅ 只做格式对齐和信息补全，不修改原文任何内容
- ✅ 多模态调用使用批量并行，保证处理效率
