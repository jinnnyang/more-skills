# Guide 3: 视觉与排版优化指南 (Visual & Formatting Guide)

本指南指导大模型在文字稿撰写完毕后，如何进行第二阶段的“视觉、排版与多媒体节奏”精修，确保最终排版与原作神似。

---

## 视觉与排版精修步骤

### 第一步：排版节奏与高亮审查 (Text Formatting Review)
根据风格配置文件的 **2.3 排版与视觉表现**，对文字草稿进行全面重排：
1. **加粗高亮控制**：
   - 检查加粗（`**双星号**`）的出现位置与密度。
   - 必须严格遵循 [视觉高亮原则]。例如：仅加粗“核心数据”、“政府部门”或“核心观点”，每段禁止加粗超过 1-2 处。
2. **段落物理隔离**：
   - 检查是否有长句或大段未换行文本。
   - 按照原作的段落密度，将文字切分为短句与小段，段落间留有空行（或使用配置中指定的分割线如 `---`、`■` 等）。

---

### 第二步：视觉占位符替换 (Image & Action Annotation Polish)
定位文字稿中所有的 `[IMAGE_PLACEHOLDER: xxx]` 标记，并将其替换为标准的 Markdown 图片格式。

**核心新增规则：Action Annotation (行动注释)**
为了防止生成的文档出现“假链接破图”，在每个 Markdown 图片标签的上方，必须紧跟一行 HTML 注释，以指导后续的 Agent 或开发者如何处理该图片：

1. **对于 `[IMAGE_PLACEHOLDER: RETAIN]`**：
   - 必须保留原始素材的 URL，不改变其内容。
   - **格式**：
     ```markdown
     <!-- ACTION: RETAIN_ORIGINAL -->
     ![白描描述](/path/to/original-image.png "图注")
     ```

2. **对于 `[IMAGE_PLACEHOLDER: GENERATE]`**：
   - 撰写一段用于文生图（Image Gen AI）的详细英文 Prompt。
   - **格式**：
     ```markdown
     <!-- ACTION: GENERATE_IMAGE | Prompt: [此处由大模型编写用于文生图的详细英文 Prompt] -->
     ![白描描述](/path/to/generated-mock-url.png "图注")
     ```

3. **对于 `[IMAGE_PLACEHOLDER: SEARCH]`**：
   - 撰写用于在搜索引擎中查找真实事实图片的关键词（Query）。
   - **格式**：
     ```markdown
     <!-- ACTION: SEARCH_WEB | Query: [此处由大模型编写用于搜索引擎找图的具体关键词] -->
     ![白描描述](/path/to/searched-mock-url.png "图注")
     ```

**基础约束规则**：
1. **配图类型匹配 (Visual Type Match)**：确保图中所描述的景象类型完全符合风格配置的 [视觉类型规范]（如搞笑综艺截图、数据图表或意境插图）。
2. **Alt Text 的细致白描 (Alt Text Description)**：Alt Text（方括号内的部分）**绝对禁止简单敷衍**。必须进行**细致的白描描述**。
3. **图注口吻对齐 (Caption Tone)**：圆括号最后用双引号括起来的图注（Caption）必须对齐配置中的 [Alt Text 与图注风格]。

---

### 第三步：负向排版清单二次过滤
检查排版是否踩中了 **3. 负向约束清单与反模式** 中的格式禁用项：
- [x] 是否存在三级以上的多级嵌套标题？
- [x] 是否无意中使用了感叹号、波浪号等禁用符号？
- [x] 是否存在大段密集文字造成视觉压迫感？

---

### 第四步：附加声明交互确认 (Declarations & Disclaimers Check)
在准备输出最终排版完成的文档前，**必须暂停**，主动调用 `ask_question` 工具（设置 `is_multi_select: true`），向用户询问是否需要在文末添加合规及版权申明。

此时提问的具体格式如下：
> **Question**: 文章已初步生成。在最终交付前，请问是否需要在文末为您生成并附加以下声明？（可多选）
> **Options**:
> - 版权与作者申明（如未经授权禁止转载等）
> - 素材来源与 AI 申明（声明图片或部分内容由生成式 AI 生成）
> - 免责与风险提示申明（如：本文为公开资料整理，不构成投资建议）
> - 争议性内容说明
> - 不需要附加任何申明

*收到用户回复后，请在文档最末尾，以契合当前文章文风（或严谨公文风）的口吻，补齐这些选中的声明。*

---

### 第五步：输出最终校验 (Final Self-Check)
对照风格配置文件中的 **4. 校验清单与防漂移机制**，进行全文最终扫描。确保所有清单项均为 `[x]` 状态，且已正确附加用户要求的声明后，交付最终文档。
