# 智能体造人术 (Role Creator) 🤖✨

一个前沿的元认知技能，旨在为专业 AI 智能体（Agent）设计、诊断并编译出结构化、低耦合的 7 维配置文件包。

---

## 📖 核心哲学

不要将大语言模型（LLM）仅仅视为一个简单的“长提示词接收器”，`role-creator` 倡导将智能体设计为一个**“岗位容器”**。一个生产级的智能体应当由模块化、高度解耦的七维架构定义，而不是拼凑而成的庞大 Prompt：

1. **`SOUL.md` (精神宪法)**：底层价值观操守、过滤机制以及核心理念冲突之间的“内在张力 (Tensions)”。
2. **`IDENTITY.md` (人设与红线)**：外在化身、交互调性、开场白以及绝对不碰的边界红线。
3. **`AGENTS.md` (SOP 与前检清单)**：Step-by-Step 标准作业程序，以及完备的前检清单 Pre-flight Checklists (A 类突出防乱码与格式校验；B 类突出排版与审美校验)。
4. **`PERSONALITY.md` (工作性格)**：为了适配不同生产岗位的可选性格偏好描述，以低消耗、高成功率和调性为第一目标。
5. **`TOOLS.md` (本地工具链)**：绑定到物理文件系统的细粒度、正交的 CLI 命令行工具与执行规范。
6. **`WIKI.md` (编译型知识网络)**：基于 Karpathy-style “停止检索，开始编译” 哲学，设计出的实体词汇、硬性规则与质量度量大纲。
7. **`BOOTSTRAP.md` 与 `HEARTBEAT.md` (运行节拍)**：智能体首次启动认证配置指引以及定时/周期心跳轮询任务列表。

---

## 🛠️ 核心功能

* **Phase 1: Grill-Me 属性深度诊断**：沿用单步提问推荐制交互，诊断上下游管道及工具链设计，精准划分智能体类别 (A 类：确定性功能体 vs. B 类：创意生成体)。
* **7 维配置文件自动生成**：在 `profiles/<role-name>/` 目录下自动化构建并写入模块化的架构配置文件。
* **Karpathy-style 知识库编译 SOP**：建立支持增量升级的 WIKI 大纲，指导智能体如何在运行报错或发现新模式时将知识编译写回 `WIKI.md` 并提交版本控制。
* **增量优化 (Diff Patches)**：支持对已有的智能体配置包进行聚焦性优化，以 Git-Diff 的局部增量补丁修改配置，而不覆盖整个文件。

---

## 📂 生成的配置结构

创建一个角色（例如：`wechat-author`）后，将在当前工作区生成以下目录树：
```text
profiles/wechat-author/
├── SOUL.md          # 价值观与决策张力
├── IDENTITY.md      # 外在设定与红线边界
├── AGENTS.md        # SOP 作业流程与前检 checklist
├── PERSONALITY.md   # 可选工作性格偏好 (例如：务实执行者 vs 视觉守护者)
├── TOOLS.md         # 本地命令行工具与正交技能接口设计
├── WIKI.md          # 实体概念、硬限制、质量度量大纲及知识编译 SOP
├── BOOTSTRAP.md     # 首次运行启动凭证配置
└── HEARTBEAT.md     # 周期性定时任务轮询列表
```

## 🚀 使用指南

### 1. 创建全新角色
在聊天框中直接发送触发词：
> **触发词**：`造人`、`做个角色`、`创建智能体`、`role-creator`、`做个智能体`、`生成角色`

**示例指令：**
```bash
/role-creator translator
```

### 2. 增量优化已有角色
输入具体的优化痛点或反馈执行日志错误：
```bash
优化 wechat-author 角色的前检清单，过滤掉原始研究笔记中的空白行。
```
技能将扫描该角色已有的全部配置文件，诊断痛点，并输出推荐的 Markdown Diff，经确认后精准打入补丁。

---

## 🧪 自动化评测 (Evals)

评测文件位于 `evals/` 目录下。可通过 `/skill-creator` 机制快速运行：

```bash
# 启动测试集校验
python scripts/run_evals.py --skill role-creator
```
评测结束后，可通过每个测试输出文件夹下的 `grading.json` 校验通过率与诊断证据。
